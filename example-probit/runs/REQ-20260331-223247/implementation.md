## Design Decisions

1. **R wrapper pattern**: Used `@name` + `NULL` roxygen blocks in separate R files for documentation, with auto-generated `RcppExports.R` providing the actual `.Call` bindings. This avoids duplicate function definitions and is the standard Rcpp + roxygen2 pattern.

2. **Hessian computation (MLE)**: Used the sqrt-weighted cross-product trick (`Xw = X.each_col() % sqrt(d); XDX = Xw.t() * Xw`) to avoid forming the N x N diagonal matrix, as specified.

3. **Numerical guards**:
   - lambda_i: falls back to |q_eta_i| when Phi_i < 1e-300 (asymptotic approximation)
   - rtruncnorm: clamps p to [1e-15, 1-1e-15] before qnorm; returns midpoint when probability mass < 1e-300
   - All log-likelihood terms use safe_log_Phi (R::pnorm with log_p=1), never log(Phi(x))

4. **MH initialization**: Calls probit_mle C++ function directly (not through R) when init is NULL, as specified.

5. **Forward declaration in probit_mh.cpp**: Declared probit_mle with its full signature to allow direct C++ call without a separate header, since probit_mle is defined in a different translation unit.

6. **d_vec abs guard (MLE)**: Applied `arma::abs(d_vec)` before sqrt to handle edge cases where d_i might be slightly negative due to floating-point arithmetic (d_i = lambda_i * (lambda_i + q_i * eta_i) is theoretically always positive but can drift).

## Smoke Test Results

With N=500, true beta = (-1, 0.5):

- **MLE**: coefficients = (-1.0877, 0.3899), matches glm() within 1.6e-7
- **Gibbs** (3500 iter, 500 burn-in): posterior mean = (-1.0845, 0.3879)
- **MH** (10000 iter, 2000 burn-in, scale=1.0): posterior mean = (-1.0899, 0.3922), acceptance rate = 0.70

All three methods produce consistent estimates. MH acceptance rate of 0.70 with default scale=1.0 is within reasonable range; the acceptance criteria target of 20-50% can be achieved by adjusting the scale parameter (e.g., scale ~2.0-2.5 for this design matrix).

---

## Known Limitations

- MH acceptance rate with default scale=1.0 may exceed the 20-50% target range for some design matrices. The scale parameter is user-tunable.
- No convergence diagnostics (effective sample size, R-hat) are computed internally; these can be added by users with the coda package on the returned draws.
