## 3. Shared Utilities — src/utils.cpp

### 3.1 Header

```cpp
#include <RcppArmadillo.h>
// [[Rcpp::depends(RcppArmadillo)]]
```

All utility functions are internal (not exported to R). They are declared in a header or within the same translation unit as needed. Use `#include "utils.h"` pattern or define in a shared header `src/utils.h`.

### 3.2 safe_log_Phi(double x)

Computes log(Phi(x)) in log-space to avoid underflow for large negative x.

```
double safe_log_Phi(double x):
  return R::pnorm(x, 0.0, 1.0, 1, 1)
```

Arguments to R::pnorm: (x, mean=0, sd=1, lower_tail=1, log_p=1).

### 3.3 safe_Phi(double x)

Computes Phi(x) directly.

```
double safe_Phi(double x):
  return R::pnorm(x, 0.0, 1.0, 1, 0)
```

Arguments: (x, mean=0, sd=1, lower_tail=1, log_p=0).

### 3.4 rtruncnorm(double mu, double lo, double hi)

Samples from a truncated normal TN(mu, 1; lo, hi) using inverse-CDF method.

Algorithm:
1. Compute p_lo = Phi(lo - mu), p_hi = Phi(hi - mu).
2. Draw u ~ Unif(0, 1).
3. Compute p = p_lo + u * (p_hi - p_lo).
4. Return mu + Phi^{-1}(p), where Phi^{-1} is R::qnorm(p, 0.0, 1.0, 1, 0).

Edge case handling:
- If p_hi - p_lo is extremely small (< 1e-300), fall back to returning the midpoint (lo + hi) / 2.0 to avoid NaN from qnorm.
- Clamp p to [1e-15, 1 - 1e-15] before calling qnorm to avoid +/-Inf returns.

## 4. Method 1: MLE via Newton-Raphson — src/probit_mle.cpp

### 4.1 C++ Interface

```cpp
// [[Rcpp::export]]
Rcpp::List probit_mle(const arma::mat& X, const arma::vec& y,
                      int max_iter = 100, double tol = 1e-8)
```

### 4.2 Algorithm Steps

1. **Initialize**: beta = zero vector of length k (where k = X.n_cols).

2. **Iterate** for t = 1, ..., max_iter:

   a. Compute eta = X * beta (N x 1 vector of linear predictors).

   b. Compute q_i = 2*y_i - 1 for each i. (Vector operation: q = 2*y - 1.)

   c. Compute q_eta_i = q_i * eta_i for each i (element-wise).

   d. For each i, compute:
      - log_Phi_i = safe_log_Phi(q_eta_i)
      - phi_i = R::dnorm(q_eta_i, 0.0, 1.0, 0) (standard normal PDF)
      - Phi_i = exp(log_Phi_i)  [or equivalently safe_Phi(q_eta_i)]
      - lambda_i = phi_i / Phi_i  (inverse Mills ratio)

      **Numerical note**: Compute lambda_i as phi_i / Phi_i where Phi_i = safe_Phi(q_eta_i). Guard against Phi_i = 0 by using: if Phi_i < 1e-300, set lambda_i = abs(q_eta_i) (the asymptotic approximation).

   e. Compute gradient weights: w_i = q_i * lambda_i. Form w as N x 1 vector.

   f. Compute Hessian diagonal: d_i = lambda_i * (lambda_i + q_i * eta_i). Form D as N x 1 vector (store diagonals only, not full matrix).

   g. Compute gradient: grad = X.t() * w (k x 1 vector).

   h. Compute weighted cross-product: XDX = X.t() * (D % X.each_col()) — i.e., X.t() * diagmat(D) * X. Equivalently: for each column of X, multiply element-wise by sqrt(D), then compute the cross-product of the result. (k x k matrix.)

      **Implementation**: Use `X.t() * arma::diagmat(d_vec) * X` where d_vec is the N x 1 vector of d_i values, or equivalently use weighted cross-product: `arma::mat Xw = X.each_col() % arma::sqrt(d_vec); arma::mat XDX = Xw.t() * Xw;` — the latter avoids forming the N x N diagonal matrix.

   i. Solve for Newton step: delta = arma::solve(XDX, grad). This solves (X'DX) * delta = X'w. Use `arma::solve` — do NOT compute the inverse explicitly.

   j. Update: beta_new = beta + delta.

   k. Convergence check: if arma::norm(beta_new - beta, 2) < tol, set beta = beta_new and break.

   l. Set beta = beta_new.

3. **Post-convergence computations**:

   a. Recompute XDX at the final beta (repeat steps 2a-2h with final beta).

   b. Compute vcov = arma::inv_sympd(XDX). This is (X'DX)^{-1}, the asymptotic variance-covariance matrix. Use `arma::inv_sympd` since XDX is symmetric positive definite.

   c. Compute se_j = sqrt(vcov(j,j)) for j = 0, ..., k-1.

   d. Compute final log-likelihood: loglik = sum of [y_i * safe_log_Phi(eta_i) + (1-y_i) * safe_log_Phi(-eta_i)] where eta = X * beta_final.

4. **Return** Rcpp::List with named elements:
   - `"coefficients"`: beta (arma::vec, length k)
   - `"vcov"`: vcov (arma::mat, k x k)
   - `"se"`: se (arma::vec, length k)
   - `"loglik"`: loglik (double)

### 4.3 Input Validation

- X must have at least k rows (N >= k).
- y must have exactly N elements (same as X.n_rows).
- All y values must be 0 or 1.
- max_iter must be >= 1.
- tol must be > 0.
- If validation fails, call Rcpp::stop() with a descriptive message.

## 5. Method 2: Bayesian Gibbs Sampler — src/probit_gibbs.cpp

### 5.1 C++ Interface

```cpp
// [[Rcpp::export]]
Rcpp::List probit_gibbs(const arma::mat& X, const arma::vec& y,
                        int n_iter = 3500, int burn_in = 500,
                        Rcpp::Nullable<arma::vec> beta0 = R_NilValue,
                        Rcpp::Nullable<arma::mat> Sigma0 = R_NilValue)
```

### 5.2 Algorithm Steps

1. **Set prior defaults**:
   - If beta0 is NULL: b0 = arma::zeros(k).
   - If Sigma0 is NULL: S0 = 100.0 * arma::eye(k, k).
   - Otherwise: b0 = Rcpp::as<arma::vec>(beta0), S0 = Rcpp::as<arma::mat>(Sigma0).

2. **Precompute constants** (computed once, used every iteration):
   - S0_inv = arma::inv_sympd(S0)  (k x k, prior precision)
   - XtX = X.t() * X  (k x k)
   - Sigma_tilde = arma::inv_sympd(S0_inv + XtX)  (k x k, posterior covariance)
   - L = arma::chol(Sigma_tilde, "lower")  (k x k, lower-triangular Cholesky)
   - S0_inv_b0 = S0_inv * b0  (k x 1, precomputed product)

3. **Initialize**:
   - beta = arma::zeros(k)  (k x 1 vector)
   - y_star = arma::zeros(N)  (N x 1 latent variable vector)
   - Allocate storage: beta_draws matrix of size (n_iter - burn_in) x k.

4. **Iterate** for t = 1, ..., n_iter:

   a. **Step 2 — Sample latent y***:
      For each i = 0, ..., N-1:
      - mu_i = dot(X.row(i), beta)  (scalar, linear predictor)
      - If y(i) == 1: y_star(i) = rtruncnorm(mu_i, 0.0, R_PosInf)  [truncated to (0, +inf)]
      - If y(i) == 0: y_star(i) = rtruncnorm(mu_i, R_NegInf, 0.0)  [truncated to (-inf, 0)]

   b. **Step 1 — Sample beta**:
      - beta_tilde = Sigma_tilde * (S0_inv_b0 + X.t() * y_star)  (k x 1 posterior mean)
      - z = arma::randn(k)  (k x 1 standard normal draw)
      - beta = beta_tilde + L * z  (k x 1 posterior draw)

   c. **Store**: If t > burn_in, store beta as row (t - burn_in - 1) of beta_draws.

5. **Post-processing**:
   - posterior_mean = arma::mean(beta_draws, 0).t()  (column means, k x 1)
   - posterior_sd = arma::stddev(beta_draws, 0, 0).t()  (column std devs, k x 1)

6. **Return** Rcpp::List with named elements:
   - `"beta_draws"`: beta_draws (arma::mat, (n_iter - burn_in) x k)
   - `"posterior_mean"`: posterior_mean (arma::vec, length k)
   - `"posterior_sd"`: posterior_sd (arma::vec, length k)

### 5.3 Input Validation

- X must have N >= k rows.
- y must have exactly N elements.
- All y values must be 0 or 1.
- n_iter must be > burn_in.
- burn_in must be >= 0.
- If Sigma0 is provided, it must be k x k and symmetric positive definite.
- If beta0 is provided, it must have length k.

### 5.4 R Wrapper — R/probit_gibbs.R

Roxygen documentation with:
- `@param X` Numeric matrix (N x k), including intercept.
- `@param y` Binary response vector of length N.
- `@param n_iter` Total MCMC iterations (default 3500).

- `@param beta0` Prior mean vector (default NULL for zero vector).

## 6. Method 3: Metropolis-Hastings — src/probit_mh.cpp

### 6.1 C++ Interface

```cpp
// [[Rcpp::export]]
Rcpp::List probit_mh(const arma::mat& X, const arma::vec& y,
                     int n_iter = 10000, int burn_in = 2000,
                     double scale = 1.0,
                     Rcpp::Nullable<arma::vec> beta0 = R_NilValue,
                     Rcpp::Nullable<arma::mat> Sigma0 = R_NilValue,
                     Rcpp::Nullable<arma::vec> init = R_NilValue)
```

### 6.2 Algorithm Steps

1. **Set prior defaults** (same logic as Gibbs — Section 5.2, step 1).

2. **Initialize beta**:
   - If init is provided: beta_current = Rcpp::as<arma::vec>(init).
   - Otherwise: run probit_mle(X, y) internally and use its coefficients as beta_current. (Call the C++ function directly, not through R.)

3. **Precompute proposal distribution**:
   - XtX_inv = arma::inv_sympd(X.t() * X)  (k x k)
   - proposal_cov = scale * scale * XtX_inv  (k x k)
   - L_C = arma::chol(proposal_cov, "lower")  (k x k, lower-triangular Cholesky)
   - S0_inv = arma::inv_sympd(S0)  (k x k, prior precision)

4. **Compute log-posterior function** (define as a lambda or helper):
   ```
   log_posterior(beta, X, y, b0, S0_inv):
     eta = X * beta
     log_lik = sum_i [y_i * safe_log_Phi(eta_i) + (1 - y_i) * safe_log_Phi(-eta_i)]
     diff = beta - b0
     log_prior = -0.5 * dot(diff, S0_inv * diff)
     return log_lik + log_prior
   ```

5. **Initialize**:
   - log_post_current = log_posterior(beta_current, ...)
   - accept_count = 0
   - Allocate: beta_draws matrix of size (n_iter - burn_in) x k.

6. **Iterate** for t = 1, ..., n_iter:

   a. **Propose**: z = arma::randn(k); beta_proposed = beta_current + L_C * z.

   b. **Evaluate**: log_post_proposed = log_posterior(beta_proposed, ...).

   c. **Accept/reject**:
      - log_ratio = log_post_proposed - log_post_current
      - log_u = log(R::runif(0.0, 1.0))
      - If log_u < log_ratio: accept. Set beta_current = beta_proposed, log_post_current = log_post_proposed, increment accept_count.
      - Else: reject. Keep beta_current unchanged.

   d. **Store**: If t > burn_in, store beta_current as row (t - burn_in - 1) of beta_draws.

7. **Post-processing**:
   - acceptance_rate = (double)accept_count / (double)n_iter
   - posterior_mean = arma::mean(beta_draws, 0).t()
   - posterior_sd = arma::stddev(beta_draws, 0, 0).t()

8. **Return** Rcpp::List with named elements:
   - `"beta_draws"`: beta_draws (arma::mat, (n_iter - burn_in) x k)
   - `"posterior_mean"`: posterior_mean (arma::vec, length k)
   - `"posterior_sd"`: posterior_sd (arma::vec, length k)
   - `"acceptance_rate"`: acceptance_rate (double)

### 6.3 Input Validation

- Same as Gibbs (Section 5.3).
- scale must be > 0.
- If init is provided, it must have length k.

## 7. Numerical Constraints

### 7.1 Log-Space Computation

All log-likelihood and log-posterior evaluations MUST use `safe_log_Phi` for log(Phi(x)) and `safe_log_Phi(-x)` for log(1-Phi(x)) = log(Phi(-x)). Never compute log(Phi(x)) as log(safe_Phi(x)) — this loses precision for large |x|.

### 7.2 Linear Algebra

- Use `arma::solve(A, b)` instead of `arma::inv(A) * b` wherever possible.
- Use `arma::inv_sympd(A)` when the inverse is explicitly needed and A is known symmetric positive definite.
- Use `arma::chol(A, "lower")` for Cholesky decomposition.
- For the MLE Hessian solve (step 2i), use `arma::solve(XDX, grad)`.
- For posterior covariance in Gibbs, `arma::inv_sympd` is appropriate since the result is reused in every iteration.

### 7.3 Random Number Generation

- Use `arma::randn(k)` for standard normal draws (uses R's RNG via RcppArmadillo).
- Use `R::runif(0.0, 1.0)` for uniform draws.
- Use `R::qnorm(p, 0.0, 1.0, 1, 0)` for the quantile function in rtruncnorm.
- Do NOT call `srand()`, `std::mt19937`, or any non-R RNG. All randomness must flow through R's RNG so that `set.seed()` in R controls reproducibility.

### 7.4 Convergence

- MLE: Convergence is declared when ||beta_new - beta||_2 < tol (Euclidean norm). If max_iter is reached without convergence, return the current beta and issue `Rcpp::warning("Newton-Raphson did not converge within max_iter iterations")`.
- MCMC (Gibbs, MH): No convergence check — run for the specified number of iterations.

### 7.5 Rank Conditions

- X must have full column rank for (X'X)^{-1} and (X'DX)^{-1} to exist.
- If `arma::solve` fails (singular system), catch the exception and call `Rcpp::stop("Design matrix X is rank-deficient")`.
