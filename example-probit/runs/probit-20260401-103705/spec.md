## 3. Shared Utilities (`src/utils.cpp`)

### 3.1 `safe_log_Phi`

```
double safe_log_Phi(double x);
```

- Computes log(Phi(x)) in a numerically stable way
- Implementation: `R::pnorm(x, 0.0, 1.0, 1, 1)` -- this is R's pnorm with lower.tail=TRUE, log.p=TRUE
- This avoids computing Phi(x) and then taking log, which loses precision for extreme x

### 3.2 `safe_Phi`

```
double safe_Phi(double x);
```

- Computes Phi(x) = standard normal CDF
- Implementation: `R::pnorm(x, 0.0, 1.0, 1, 0)` -- lower.tail=TRUE, log.p=FALSE

### 3.3 `rtruncnorm`

```
double rtruncnorm(double mu, double lo, double hi);
```

- Samples from TN(mu, 1; lo, hi) using inverse CDF method
- Algorithm:
  1. Compute p_lo = Phi(lo - mu) and p_hi = Phi(hi - mu) using `R::pnorm`
  2. Handle boundary cases: if lo = -inf, p_lo = 0; if hi = +inf, p_hi = 1
  3. Draw u ~ Unif(0, 1) via `R::runif(0.0, 1.0)`
  4. Compute p = p_lo + u * (p_hi - p_lo)
  5. Return mu + Phi^{-1}(p) via `R::qnorm(p, 0.0, 1.0, 1, 0)`
- Numerical guard: clamp p to [1e-15, 1 - 1e-15] before calling qnorm to avoid returning +/-Inf
- Use R_PosInf and R_NegInf for infinite bounds

All three utility functions must have `#include <RcppArmadillo.h>` and use `// [[Rcpp::depends(RcppArmadillo)]]`.

## 4. Method 1: MLE via Newton-Raphson (`src/probit_mle.cpp`)

### 4.1 C++ Interface

```cpp
// [[Rcpp::export]]
Rcpp::List probit_mle(const arma::mat& X, const arma::vec& y,
                      int max_iter = 100, double tol = 1e-8);
```

**Returns** an `Rcpp::List` with named elements:
- `coefficients`: arma::vec of length k -- the MLE estimate beta_hat
- `vcov`: arma::mat of dimension k x k -- the variance-covariance matrix (X'DX)^{-1}
- `se`: arma::vec of length k -- standard errors, SE_j = sqrt(vcov_{jj})
- `loglik`: double -- the log-likelihood at convergence l(beta_hat)
- `iterations`: int -- number of iterations used
- `converged`: bool -- whether convergence criterion was met

### 4.2 Algorithm Steps

1. **Initialize**: beta = zeros(k)
2. **Iterate** for t = 1, ..., max_iter:
   a. Compute eta = X * beta (N-vector of linear predictors)
   b. Compute q_i = 2*y_i - 1 for each i
   c. For each i, compute:
      - lambda_i = phi(q_i * eta_i) / Phi(q_i * eta_i)
      - Use: numerator = R::dnorm(q_i * eta_i, 0, 1, 0), denominator = safe_Phi(q_i * eta_i)
      - Guard: if denominator < 1e-300, set lambda_i = |q_i * eta_i| (asymptotic approximation)
   d. Compute w_i = q_i * lambda_i (N-vector)
   e. Compute d_i = lambda_i * (lambda_i + q_i * eta_i) (N-vector, all entries > 0)
   f. Form gradient: g = X.t() * w (k-vector)
   g. Form Hessian diagonal D as N-vector d, then compute H = X.t() * diagmat(d) * X (k x k matrix)
   h. Solve H * step = g via `arma::solve(H, g)` -- do NOT explicitly invert H
   i. Update: beta = beta + step
   j. Check convergence: if arma::norm(step) < tol, set converged = true and break
3. **Compute output**:
   a. vcov = arma::inv_sympd(H) -- H = X'DX is symmetric positive definite at convergence
   b. se_j = sqrt(vcov(j,j)) for each j
   c. Compute loglik = sum_i [ y_i * safe_log_Phi(eta_i) + (1-y_i) * safe_log_Phi(-eta_i) ]
      - Note: log(1 - Phi(x)) = log(Phi(-x)) = safe_log_Phi(-x) by symmetry

### 4.3 Numerical Stability Requirements

- Use `safe_log_Phi` for all log-CDF computations -- never compute log(Phi(x)) directly
- Use `arma::solve` for the Newton step, not matrix inversion
- Use `arma::inv_sympd` only for the final variance-covariance matrix (after convergence)
- Guard lambda_i computation against Phi(q_i * eta_i) being numerically zero
- The diagonal d_i must be strictly positive; if any d_i <= 0 due to numerical issues, clamp to a small positive value (1e-10)

## 5. Method 2: Bayesian Gibbs Sampler (`src/probit_gibbs.cpp`)

### 5.1 C++ Interface

```cpp
// [[Rcpp::export]]
Rcpp::List probit_gibbs(const arma::mat& X, const arma::vec& y,
                        int n_iter = 3500, int burn_in = 500,
                        Rcpp::Nullable<arma::vec> beta0 = R_NilValue,
                        Rcpp::Nullable<arma::mat> Sigma0 = R_NilValue);
```

**Returns** an `Rcpp::List` with named elements:
- `beta_draws`: arma::mat of dimension (n_iter - burn_in) x k -- post-burn-in posterior draws (each row is one draw)
- `posterior_mean`: arma::vec of length k -- column means of beta_draws
- `posterior_sd`: arma::vec of length k -- column standard deviations of beta_draws

### 5.2 Algorithm Steps

1. **Set prior defaults**: if beta0 is NULL, set beta0 = zeros(k). If Sigma0 is NULL, set Sigma0 = 100 * eye(k).
2. **Precompute** (constant quantities -- compute once):
   a. Sigma0_inv = arma::inv_sympd(Sigma0) (k x k)
   b. XtX = X.t() * X (k x k)
   c. Sigma_tilde = arma::inv_sympd(Sigma0_inv + XtX) (k x k)
   d. L = arma::chol(Sigma_tilde, "lower") -- lower-triangular Cholesky factor
   e. Sigma0_inv_beta0 = Sigma0_inv * beta0 (k-vector, precompute)
3. **Initialize**: beta = zeros(k), y_star = zeros(N)
4. **Allocate storage**: arma::mat beta_store of dimension (n_iter - burn_in) x k
5. **Iterate** for t = 1, ..., n_iter:
   a. **Step 2 -- Sample latent y***: For each i = 1, ..., N:
      - mu_i = dot(X.row(i), beta)
      - If y_i == 1: y_star_i = rtruncnorm(mu_i, 0.0, R_PosInf)
      - If y_i == 0: y_star_i = rtruncnorm(mu_i, R_NegInf, 0.0)
   b. **Step 1 -- Sample beta**:
      - beta_tilde = Sigma_tilde * (Sigma0_inv_beta0 + X.t() * y_star) (k-vector)
      - Draw z ~ N(0, I_k) via arma::randn(k)
      - beta = beta_tilde + L * z
   c. **Store**: If t > burn_in, store beta as row (t - burn_in - 1) of beta_store
6. **Compute output**:
   a. posterior_mean = arma::mean(beta_store, 0).t() -- column means
   b. posterior_sd = arma::stddev(beta_store, 0, 0).t() -- column standard deviations

### 5.3 Numerical Stability Requirements

- Sigma_tilde must be computed via `inv_sympd` since (Sigma0_inv + X'X) is symmetric positive definite
- Cholesky decomposition must use the lower-triangular convention (`"lower"`)
- The rtruncnorm utility handles the truncated normal edge cases

## 6. Method 3: Metropolis-Hastings MCMC (`src/probit_mh.cpp`)

### 6.1 C++ Interface

```cpp
// [[Rcpp::export]]
Rcpp::List probit_mh(const arma::mat& X, const arma::vec& y,
                     int n_iter = 10000, int burn_in = 2000,
                     double scale = 1.0,
                     Rcpp::Nullable<arma::vec> beta0 = R_NilValue,
                     Rcpp::Nullable<arma::mat> Sigma0 = R_NilValue,
                     Rcpp::Nullable<arma::vec> init = R_NilValue);
```

**Returns** an `Rcpp::List` with named elements:
- `beta_draws`: arma::mat of dimension (n_iter - burn_in) x k -- post-burn-in posterior draws
- `posterior_mean`: arma::vec of length k
- `posterior_sd`: arma::vec of length k
- `acceptance_rate`: double -- fraction of proposals accepted (over all n_iter iterations)

### 6.2 Algorithm Steps

1. **Set prior defaults**: if beta0 is NULL, set beta0 = zeros(k). If Sigma0 is NULL, set Sigma0 = 100 * eye(k).
2. **Set initial value**: if init is NULL, obtain MLE estimate by calling `probit_mle(X, y)` and extracting the coefficients. Otherwise use the provided init vector.
3. **Precompute** (constant quantities):
   a. Sigma0_inv = arma::inv_sympd(Sigma0) (k x k)
   b. XtX_inv = arma::inv_sympd(X.t() * X) (k x k)
   c. proposal_cov = scale * scale * XtX_inv (k x k)
   d. L_C = arma::chol(proposal_cov, "lower") -- lower-triangular Cholesky of proposal covariance
   e. Sigma0_inv_beta0 = Sigma0_inv * beta0 (k-vector)
4. **Define log-posterior function** (internal helper or inline):

   ```
   log_posterior(beta, X, y, Sigma0_inv, Sigma0_inv_beta0):
     eta = X * beta
     ll = sum_i [ y_i * safe_log_Phi(eta_i) + (1 - y_i) * safe_log_Phi(-eta_i) ]
     diff = beta - beta0  // but we precomputed Sigma0_inv_beta0
     log_prior = -0.5 * dot(diff, Sigma0_inv * diff)
     return ll + log_prior
   ```

   Note: The prior term can be computed as -0.5 * (dot(beta, Sigma0_inv * beta) - 2 * dot(beta, Sigma0_inv_beta0) + dot(beta0, Sigma0_inv_beta0)). The last term is constant and cancels in the MH ratio, so it can be omitted from log_posterior.

5. **Initialize**: beta_current = init value from step 2. Compute log_post_current = log_posterior(beta_current, ...). Set n_accept = 0.
6. **Allocate storage**: arma::mat beta_store of dimension (n_iter - burn_in) x k
7. **Iterate** for t = 1, ..., n_iter:
   a. Draw z ~ N(0, I_k) via arma::randn(k)
   b. Propose: beta_prop = beta_current + L_C * z
   c. Compute log_post_prop = log_posterior(beta_prop, ...)
   d. log_r = log_post_prop - log_post_current
   e. Draw log_u = log(R::runif(0.0, 1.0))
   f. If log_u < log_r: accept -- set beta_current = beta_prop, log_post_current = log_post_prop, increment n_accept
   g. Else: reject -- beta_current unchanged
   h. If t > burn_in: store beta_current as row (t - burn_in - 1) of beta_store
8. **Compute output**:
   a. posterior_mean = arma::mean(beta_store, 0).t()
   b. posterior_sd = arma::stddev(beta_store, 0, 0).t()
   c. acceptance_rate = (double)n_accept / (double)n_iter

### 6.3 Numerical Stability Requirements

- Use `safe_log_Phi` for all log-CDF computations in the log-posterior
- Compute acceptance on log scale: compare log(u) to log(r), never exponentiate
- If log_post_prop returns -Inf (e.g., beta_prop leads to numerically zero likelihood), always reject
- The proposal Cholesky L_C uses `arma::chol` with `"lower"` convention

## 7. R Wrapper Functions

### 7.1 `R/probit_mle.R`

```r
#' Probit MLE via Newton-Raphson
#'
#' Estimates the probit model by maximum likelihood using
#' the Newton-Raphson algorithm implemented in C++.
#'
#' @param X Numeric matrix (N x k) of covariates. Must include an
#'   intercept column if an intercept is desired.
#' @param y Numeric vector of length N with binary outcomes (0 or 1).
#' @param max_iter Integer. Maximum number of Newton-Raphson iterations
#'   (default 100).
#' @param tol Double. Convergence tolerance on the norm of the step
#'   (default 1e-8).
#'
#' @return A list with components:
#' \describe{
#'   \item{coefficients}{Numeric vector of estimated coefficients.}
#'   \item{vcov}{Variance-covariance matrix of the estimates.}
#'   \item{se}{Standard errors of the estimates.}
#'   \item{loglik}{Log-likelihood at convergence.}
#'   \item{iterations}{Number of iterations performed.}
#'   \item{converged}{Logical indicating convergence.}
#' }
#'
#' @examples
#' set.seed(42)
#' N <- 200
#' X <- cbind(1, rnorm(N))
#' beta_true <- c(-1, 0.5)
#' y <- rbinom(N, 1, pnorm(X %*% beta_true))
#' fit <- probit_mle(X, y)
#' fit$coefficients
#'
#' @export
probit_mle <- function(X, y, max_iter = 100L, tol = 1e-8) {
    # Input validation
    X <- as.matrix(X)
    y <- as.numeric(y)
    stopifnot(is.matrix(X), is.numeric(y))
    stopifnot(nrow(X) == length(y))
    stopifnot(all(y %in% c(0, 1)))
    stopifnot(max_iter >= 1L, tol > 0)

    .Call(`_exampleProbit_probit_mle`, X, y, as.integer(max_iter), tol)
}
```

Note: The `.Call` line above is illustrative. In practice, Rcpp generates the glue code automatically from the `// [[Rcpp::export]]` annotations. Builder should simply export the R function that calls the C++ function directly. The actual wrapper should just call the C++ function by name (Rcpp handles the rest). A simpler pattern:

```r
probit_mle <- function(X, y, max_iter = 100L, tol = 1e-8) {
    X <- as.matrix(X)
    y <- as.numeric(y)
    stopifnot(nrow(X) == length(y), all(y %in% c(0, 1)))
    probit_mle_cpp(X, y, as.integer(max_iter), tol)  # calls the Rcpp-exported function
}
```

If the C++ function name is `probit_mle`, there will be a name collision. Either:
- Name the C++ function `probit_mle_cpp` and export it, then wrap in R as `probit_mle`, OR
- Name the R wrapper `probit_mle` and have it call the C++ function directly (which Rcpp exports as `probit_mle` in the NAMESPACE). In this case, the R wrapper file is optional -- Rcpp handles the export.

**Recommended approach**: Let the C++ `// [[Rcpp::export]]` create the R-callable function. Add a thin R wrapper only for input validation. Name C++ exports `probit_mle_cpp`, `probit_gibbs_cpp`, `probit_mh_cpp` to avoid collision, then have R wrappers `probit_mle`, `probit_gibbs`, `probit_mh` call them.
