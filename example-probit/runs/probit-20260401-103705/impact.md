# Impact Assessment

## Request Summary

Greenfield R package build: `exampleProbit` — three probit estimation methods in C++ via Rcpp/RcppArmadillo, plus a Monte Carlo simulation study comparing all three.

## Workflow Selection

**Workflow 11 (Simulation + Code)**: New estimator implementations + Monte Carlo evaluation.
- Builder: implements the R package structure and C++ estimators
- Simulator: implements the DGP and simulation harness
- Tester: validates both unit tests and simulation results
- All three dispatched in parallel after planner produces specs

## Target Repository State

Empty repository (no files). This is a **full package creation** — no existing code to preserve or refactor.

## Risk Areas

| Risk | Mitigation |
| --- | --- |
| Numerical stability in probit likelihood at extreme values | Use R::pnorm log-space computation; spec requires safe_log_Phi |
| Truncated normal sampling edge cases | Inverse-CDF method with proper bounds handling |
| MH acceptance rate tuning | Scale parameter s=1 with warm start from MLE |
| CRAN compliance for Rcpp packages | Follow r-package profile CRAN checklist |
| Compilation on macOS (clang + Armadillo) | Standard Makevars with PKG_LIBS |
