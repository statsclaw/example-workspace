## Design Choices

1. **Default values in header only**: C++ default arguments are specified in `fect.h` declarations (not in `.cpp` definitions) to avoid redefinition errors. This is standard C++ practice.

2. **Warm-start reset on burn-in restart**: When `fe_ad_inter_iter` and `fe_ad_inter_covar_iter` restart after the burn-in phase converges, the warm-start factors are reset (F_prev.reset(), L_prev.reset()) since the iteration resets to Y0.

3. **SSR check is secondary**: The SSR-based convergence is an additional early-exit path that does not replace the Frobenius-norm check. It only triggers when BOTH the SSR is essentially flat AND the Frobenius metric is within 10x of tolerance.

4. **Joint convergence in beta_iter**: The original absolute-norm convergence check is replaced with a joint relative-norm check that monitors both beta change and factor change. This prevents the situation where beta appears converged but factors are still moving.

5. **Subspace iteration count fixed at 2**: Two power iterations provide a good balance between computational cost and subspace quality. When the subspace is already converged, they have negligible cost. When far from converged, they bring it close enough for the outer EM loop.

## Known Limitations

- The `ife()` and `ife_part()` Rcpp-generated R wrappers now require F_prev and L_prev arguments (Rcpp cannot parse `arma::mat()` defaults). These R wrappers are never called directly by R code -- they are only used internally from C++, where the default values in `fect.h` apply correctly.

- The warm-start in cfe_iter does not track factors across burn-in restarts (same as fe_ad_inter_iter).
